性能优化基础

前端性能优化是现代 Web 开发中的核心技能之一。一个性能良好的网站不仅能提供更好的用户体验，还能提升 SEO 排名、降低服务器成本，并提高用户留存率。

为什么需要性能优化？

用户体验影响

  • 加载速度：页面加载时间每增加 1 秒，转化率就会下降约 7%
  • 交互响应：流畅的交互体验能显著提升用户满意度
  • 移动端体验：移动设备的性能限制使得优化更加重要

业务影响

  • 搜索引擎排名：Google 等搜索引擎将页面速度作为排名因素
  • 用户留存：慢速网站会导致用户流失
  • 服务器成本：优化可以减少服务器负载和带宽消耗

性能指标

Core Web Vitals

Google 提出的核心 Web 指标：

  1. LCP (Largest Contentful Paint)

    • 最大内容绘制时间
    • 理想值：< 2.5 秒

  2. FID (First Input Delay)

    • 首次输入延迟
    • 理想值：< 100 毫秒

  3. CLS (Cumulative Layout Shift)

    • 累积布局偏移
    • 理想值：< 0.1

其他重要指标

  • TTFB (Time to First Byte)：首字节时间
  • FCP (First Contentful Paint)：首次内容绘制
  • TTI (Time to Interactive)：可交互时间

性能优化原则

1. 测量优先

在优化之前，必须先测量：

// 使用 Performance API 测量性能
const perfData = window.performance.timing;
const pageLoadTime = perfData.loadEventEnd - perfData.navigationStart;
console.log('页面加载时间:', pageLoadTime, 'ms');

2. 优化关键路径

关注影响用户体验的关键资源：

  • HTML 文档
  • CSS 样式表
  • 关键 JavaScript
  • 首屏图片

3. 渐进式优化

  • 先优化影响最大的部分
  • 使用 80/20 原则
  • 持续监控和改进

优化策略概览

代码层面

  • 代码分割和懒加载
  • 减少 JavaScript 执行时间
  • 优化 CSS 选择器
  • 使用现代 JavaScript 特性

资源层面

  • 图片优化和格式选择
  • 字体加载优化
  • 资源压缩和合并
  • CDN 使用

网络层面

  • HTTP/2 和 HTTP/3
  • 缓存策略
  • 预加载和预连接
  • 减少请求数量

工具推荐

性能分析工具

  1. Chrome DevTools

    • Performance 面板
    • Lighthouse 审计
    • Network 面板

  2. WebPageTest

    • 多地点测试
    • 详细的性能报告

  3. Lighthouse

    • 自动化性能审计
    • 提供优化建议

监控工具

  • Google Analytics
  • New Relic
  • Sentry

总结

性能优化是一个持续的过程，需要：

  1. 建立性能预算：设定明确的目标指标
  2. 持续监控：定期检查性能指标
  3. 迭代优化：根据数据不断改进
  4. 团队协作：将性能作为开发流程的一部分
