
# 标签命名规范

## 📋 标签命名规则

### 1. 技术标签
- **前端技术**：`html`, `css`, `javascript`, `vue`, `react`, `angular`
- **后端技术**：`nodejs`, `python`, `java`, `php`, `go`
- **数据库**：`mysql`, `mongodb`, `redis`, `postgresql`
- **工具**：`git`, `docker`, `jenkins`, `vscode`

### 2. 主题标签
- **学习笔记**：`学习笔记`, `教程`, `入门`
- **项目实践**：`项目实战`, `案例分析`, `最佳实践`
- **问题解决**：`bug修复`, `问题排查`, `解决方案`
- **工具使用**：`工具推荐`, `使用技巧`, `配置指南`

### 3. 难度标签
- **初级**：`入门`, `基础`, `新手`
- **中级**：`进阶`, `中级`, `实战`
- **高级**：`高级`, `专家`, `源码`

### 4. 类型标签
- **技术文章**：`技术分享`, `开发经验`, `编程技巧`
- **学习记录**：`学习笔记`, `读书笔记`, `课程总结`
- **项目展示**：`项目展示`, `作品集`, `开源项目`
- **生活感悟**：`生活感悟`, `思考`, `随笔`

## 🎯 标签使用建议

1. **数量控制**：每篇文章建议使用 3-8 个标签
2. **层次分明**：使用不同层次的标签（技术+主题+难度）
3. **中英文**：技术标签用英文，主题标签用中文
4. **一致性**：相同含义的标签保持统一命名

## 📝 示例

```yaml
---
title: Vue 3 组合式 API 入门教程
date: 2024-01-15
tags:
  - vue
  - javascript
  - 前端开发
  - 入门教程
  - 组合式API
categories:
  - 前端技术
---
```
